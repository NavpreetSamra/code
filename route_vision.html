class route_vision.astar.AStar(graph=None, meshSize=(3, 3), plots=None, velocity_type='harmonic', velocity_attrs={'ya': 3.0, 'scalar': 10, 'xa': 3.0, 'xf': 2.0, 'yf': 2.0}, heur={'comp': 'vel', 'metric': 'octile', 'alpha': 1.0}, heur_weight=5, embedPath=None, embedObstacle=None, auto=True, links=8)[source]¶

Bases: route_vision.grid.Grid

AStar class, for path search

Subclassed from route_vision.grid.Grid for ease of use and to enable faster iteration.
AStar will function without loss supplying withou any kwargs for Grid if a networkx.Graph() with node ids defined as position, edge weight velocity, and a local velocity field for each node is supplied

For more information please see docs.

Parameters:
  • graph (networkx.Graph) – option to supply graph. If graph not supplied graph will be built from Grid attributes
  • meshSize (tuple.int) – x,y size of graph
  • velocity_type (str) – type of profile, scalar or harmonic
  • velocity attrs (dict) – define velocity profile
  • heur (dict) – define heuristic type
  • heur_weight (float) – define relative weight of prioritization (cost + heur*estimate)
  • embedPath (tuple.float) – specify value of location to set velocity
  • embedObstacle (bool) – embed an obstacle
  • auto (bool) – auto generate profile and build attrs
  • links (int) – if auto generating graphs, specify if graph is manhattan or octile
