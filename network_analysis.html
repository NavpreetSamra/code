class network_analysis.network_analysis.SystemNetwork(nodeMap, macMap, mesh, links)[source]¶

Bases: object

Class for identifying systems of networks from sets of MAC address peer links

SystemNetwork takes mappings between nodes via MAC addresses and the link’s quality and creates clusters of networks by partitioning nodes via walking through the mesh. Then SystemNetwork evaluates each network to ensure each node has a quality link with each other node in its network. Networks are attributed True(paths exists with quality links to/from each node) or False. If there is only 1 node in a network it is recorded as not a network and a warning indicating such is reported.

Parameters:
  • nodeMap (dict) – map from MAC to node id {MAC: Node} ; {str: str}
  • macMap (dict) – map from node id to MACs {Node: [MAC1, MAC2]} ; {str: list.str}
  • mesh (dict) – MAC mesh {MAC: [MAC1, ... MACi]} ; {str, list.str}
  • links (dict) – link with quality (NB not bidirectional) from MAC1 to MAC2 {(MAC1, MAC2): val}; {tuple.str, bool}

The file peer_link_example.csv is an example file which can be loaded with the network_analysis.example_parser.parse to generate the inputs for system network

network_analysis.example_parser.parse(fileName)[source]¶

Parse csv with node id + JSON blob, links are added if they meet:

  • 2.4GHz frequency >= 8 MCS quality
  • 5GHz frequency >= 6 MCS quality

Note this parser is built to handle a specifically malformed JSON blob where both “{“”key””:”“val”“}” “{“”key””:val”“}” exist inline
